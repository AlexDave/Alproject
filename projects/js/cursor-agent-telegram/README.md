# Cursor Agent → Telegram

Локальный сервис: подключается к уже запущенному **Cursor** через **Chrome DevTools Protocol (CDP)**, периодически снимает текст из интерфейса чата/агента и шлёт обновления в **Telegram**. Сообщения боту (и кнопка «Продолжить») вводят текст в композер Cursor и отправляют его (Enter или Ctrl+Enter — настраивается).

Никаких внешних лицензий: нужны только ваш **Telegram Bot Token** и **user id**.

## Требования

- **Node.js 20+**
- **Cursor** с флагом отладки: `--remote-debugging-port=9222`
- Проверка: в браузере открывается `http://127.0.0.1:9222/json` и отдаёт JSON
- Перед опросом сервис сам запрашивает тот же URL; если порта нет — процесс завершится с подсказкой. Отключить только для особых сценариев: **`AGENT_SKIP_CDP_PROBE=1`**. Автотест логики: `npm run test` (пакет `cdp-probe`).

### Запуск Cursor (Windows)

```powershell
& "$env:LOCALAPPDATA\Programs\cursor\Cursor.exe" --remote-debugging-port=9222
```

Полностью закройте Cursor и запустите снова с этим флагом.

## Настройка

1. Скопируйте `.env.example` в `.env`.
2. **Вариант A — только Hub (портал Next.js):** задайте `HUB_URL` (например `http://localhost:3000`) и `HUB_INGEST_SECRET` — тот же секрет, что в `apps/portal/.env.local` как `HUB_INGEST_SECRET`. В портале задайте `HUB_PASSWORD` и `HUB_SESSION_SECRET`.
3. **Вариант B — Telegram:** токен от [@BotFather](https://t.me/BotFather), числовой `user id`, `TELEGRAM_ALLOWED_USER_IDS`.
4. Можно **оба канала** одновременно (Hub + Telegram).

## Запуск

```bash
cd projects/js/cursor-agent-telegram
npm install
npm run build
npm start
```

Разработка с автоперезапуском:

```bash
npm run dev
```

## Поведение

- Локальный **HTTP `/control`** на `127.0.0.1` и порту **`AGENT_CONTROL_PORT`** (по умолчанию **4000**). Если видите **EADDRINUSE** — уже запущен другой экземпляр relay (например, отдельный терминал + `npm run dev` с `cursor-agent-relay`). Закройте дубликат или задайте другой порт и **`AGENT_CONTROL_URL`** в портале.
- Фоновый опрос DOM каждые `POLL_INTERVAL_MS` мс. В **Telegram** по умолчанию режим **`TELEGRAM_NOTIFY_MODE=idle`**: сообщение отправляется **один раз**, когда текст снимка перестал меняться как минимум **`TELEGRAM_STABLE_MS`** мс (по умолчанию 16000). Текст — короткое **саммари** последнего блока чата (до **`TELEGRAM_SUMMARY_MAX_CHARS`** символов), а не весь лог. Между двумя разными такими «завершениями» действует дополнительный интервал **`MIN_NOTIFY_INTERVAL_MS`**.
- Для отладки прежнего поведения (большое сообщение при каждом изменении DOM, с ограничением по `MIN_NOTIFY_INTERVAL_MS`) задайте **`TELEGRAM_NOTIFY_MODE=legacy`**.
- **Hub** по-прежнему получает актуальный снимок на каждом успешном опросе — это не связано с режимом Telegram.
- Любой текст от разрешённого пользователя — вводится в композер активного агента (textarea или `contenteditable`) и отправляется.
- **Клавиатура в Telegram:** «Переключить агента», **«Переключить режим»**, «Новый агент». **Переключить режим** — **Shift+Tab** и ответ по **иконке (классы codicon, SVG use, aria)** и тексту контрола. Для своей «таблетки» режима можно задать **`CURSOR_COMPOSER_MODE_TRIGGER_SELECTOR`**. Под «✅ Готово» **«Выполнить план»**, если итог похож на план или в снимке есть «View Plan» (`CURSOR_PLAN_BUILD_KEYS`).
- Текст «✅ Готово» берётся из снимка так: если в хвосте чата есть блок, похожий на **план** (нумерация, маркеры, заголовок Plan/План), он попадает в Telegram целиком; иначе используется прежняя эвристика «последний блок / самый длинный хвост». Так короткий статус после плана не «вытесняет» сам план. Кодовая фраза `TELEGRAM_DONE_PHRASE` ищется в **последних N блоках** (по умолчанию `TELEGRAM_DONE_PHRASE_TAIL_BLOCKS=8`), а не только в двух, чтобы маркер не «терялся» за тулом/статусом.
- После idle-уведомления «✅ Готово», если в саммари есть **от двух до двенадцати** распознанных вариантов (нумерованный список, маркеры `•`/`-`/`*` или несколько строк с «?»), бот отправляет **опрос** (`TELEGRAM_PLAN_POLL`, по умолчанию вкл.). Опрос **неанонимный**, чтобы голос принимался только от пользователей из `TELEGRAM_ALLOWED_USER_IDS`. Выбранный вариант **отправляется в композер** Cursor как инструкция. Текст вопроса по умолчанию — `TELEGRAM_PLAN_POLL_DEFAULT_QUESTION`, если из саммари не извлечено короткое вступление до списка.
- Фразу продолжения (`CONTINUE_PHRASE`) можно ввести текстом вручную; справка — `/help`.

### Форматирование сообщений в Telegram

Разметка в [`src/telegram-html.ts`](projects/js/cursor-agent-telegram/src/telegram-html.ts): **idle** — «✅ Готово», метка агента и тело без отдельной строки «план/итог»; текст перед отправкой проходит [`stripTrailingCursorChrome`](projects/js/cursor-agent-telegram/src/telegram-summary.ts) (хвосты вида View Plan, `.plan.md` без пробела и т.п.). **После смены агента** — «Активный агент» и «Последний итог в чате». **Legacy** и **ошибки CDP** — как раньше.

## Важно про DOM

Версии Cursor меняют вёрстку (в т.ч. вкладки vs выпадающий список режима композера). Если снимки пустые или не из той панели — смотрите `src/cdp/cursor-session.ts` (список селекторов в `snapshotAgentText`) и при необходимости добавьте свои селекторы под вашу сборку.

Если отправка не срабатывает, попробуйте `SEND_MODIFIERS=mod+enter` или проверьте, что фокус в композере чата агента.

## Если бот «молчит» на /start

1. В консоли при `npm start` должна быть строка **`Telegram API: бот @...`**. Если ошибка сети — **VPN** или другой доступ к `api.telegram.org`.
2. Пишите боту в **личке**, не из группы (или отключите privacy в BotFather и добавьте бота в группу осознанно).
3. Отправьте **`/whoami`** — придёт ваш **числовой id**; сравните с `TELEGRAM_ALLOWED_USER_IDS` в `.env` на машине, где запущен `npm start`.
4. Перезапустите процесс после правок `.env`. Убедитесь, что не запущено **два** экземпляра бота с одним токеном (конфликт long polling).

### Если `api.telegram.org` открывается в браузере, но бот всё равно не отвечает

- Это частый кейс: браузер использует системный прокси/туннель, а `node` (и `grammy`) ходят напрямую и получают timeout.
- Добавьте в `.env` прокси для Telegram API:

```env
TELEGRAM_PROXY_URL=http://127.0.0.1:7890
```

- (Опционально) можно задать альтернативный корень API:

```env
TELEGRAM_API_ROOT=https://api.telegram.org
```

- После правки `.env` перезапустите relay.

Переменные из `.env` подхватываются из **каталога проекта**, а не из текущей папки в терминале.

## Docker

Сборка из каталога проекта:

```bash
docker build -t cursor-agent-telegram .
```

Cursor должен быть на **хосте**, контейнеру укажите `CDP_URL=http://host.docker.internal:9222` (Windows/Mac; на Linux см. документацию Docker про доступ к хосту).

## Безопасность

- Храните `TELEGRAM_BOT_TOKEN` только в `.env`, не коммитьте.
- Ограничивайте доступ через `TELEGRAM_ALLOWED_USER_IDS`.
- CDP даёт мощный доступ к окну приложения; не открывайте порт 9222 в интернет, используйте только localhost или доверенную сеть (например Tailscale).
