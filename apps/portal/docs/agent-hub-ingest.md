# Контракт Agent Hub Ingest

Единый формат тела **POST** для:

- `https://<portal>/api/agent-hub/ingest` (портал проксирует на шлюз, если задан `AGENT_HUB_BACKEND_URL`)
- `http://<gateway>/ingest` (сервис `@alproject/agent-hub-gateway`)

## Заголовки

- `Authorization: Bearer <HUB_INGEST_SECRET>` — обязателен; секрет должен совпадать с `HUB_INGEST_SECRET` на портале и на шлюзе.

## JSON-тело

| Поле | Тип | Обязательность | Описание |
|------|-----|----------------|----------|
| `schemaVersion` | string | нет | Версия контракта; актуальное значение: **`1`**. Старые клиенты без поля считаются совместимыми. |
| `snapshot` | string | нет | Текст снимка панели агента; по умолчанию `""`. |
| `cdpError` | string \| null | нет | Сообщение об ошибке CDP или `null`. |

Дополнительные поля допускаются и **игнорируются** парсером (Zod `.passthrough()`), чтобы расширения не ломали приём.

## Пример

```json
{
  "schemaVersion": "1",
  "snapshot": "…",
  "cdpError": null
}
```

## Ответ

Успех: `{ "ok": true }` и HTTP 200.

Ошибки: JSON с полем `error` и коды 400 / 401 / 503 по смыслу.

## Смена версии

При несовместимых изменениях поднимайте `schemaVersion` и обновляйте этот файл, портал и шлюз в одном изменении.
